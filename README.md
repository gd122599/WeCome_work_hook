  # Wecome-Work-Hook 🚀

  **企业微信 PC 端 Hook 自动化框架**

  基于 Hook 技术深度接入企业微信，提供稳定的 HTTP API 接口，支持消息实时收发、群组自动化管理，助力开发者快速构建企业微信机器人与自动化系统。

  **适配版本：企业微信 v4.1.36.6012 ｜ 系统要求：Windows 10 / 11**


  </div>

  ---

  ## 🎯 能做什么？

  > 不需要修改企业微信源码，无需抓包，注入即用。

  | 场景 | 描述 |
  |------|------|
  | 🤖 **智能客服机器人** | 实时接收消息回调，对接 AI 模型（GPT/文心等）自动回复 |
  | 📢 **消息群发推送** | 向多个群/联系人推送文本、图片、文件、小程序等消息 |
  | 👥 **群组自动化管理** | 自动建群、拉人入群、改群名/群公告、管理成员与管理员 |
  | 🔗 **系统集成对接** | 通过 Webhook 回调对接 CRM、工单、监控报警等内部系统 |

  ## ✨ 核心功能

  ### 🔐 登录与账号
  - 获取登录状态、登录二维码、个人二维码
  - 输入登录验证码、退出登录
  - 获取当前账号个人信息

  ### 📨 消息发送（12 种消息类型）
  - 文本、图片、GIF 表情、文件、视频、语音
  - 名片、小程序、视频号、卡片链接
  - 群 @ 消息（@ 指定成员）、位置消息

  ### 📥 消息接收与回调
  - 实时接收企微推送的消息（含群聊 / 单聊），自动落盘日志
  - 支持配置 **Webhook 回调地址**，收到的消息自动转发到你的业务服务

  ### 👥 群组管理
  - 创建群聊（外部群 / 内部群）、解散群聊、退出群聊
  - 邀请成员（40 人以下 / 40 人以上）、移除群成员
  - 修改群名称、发送群公告、获取群信息与群二维码
  - 添加 / 删除管理员、转让群主
  - 群欢迎语管理（列表 / 获取 / 设置 / 删除）
  - 允许/禁止修改群名、邀请确认开关

  ### 👤 联系人与好友
  - 获取联系人列表（外部）、好友列表（内部）、群列表、群成员、公司列表、部门列表
  - 添加好友（群成员 / 网络查询）、删除好友、同意好友请求
  - 设置好友备注、描述、公司、手机号
  - 本地查询好友信息、网络查询陌生人信息

  ### 🏷️ 标签管理
  - 标签组列表，标签的添加 / 修改 / 删除
  - 一个好友打多个标签、一个标签打多个好友
  - 获取标签下联系人、获取用户所有标签、移除标签下联系人

  ### 🔄 ID 转换
  - 企业用户 ID ↔ 开放平台 ID
  - 企业群 ID ↔ 开放平台 ID

  ### 📦 CDN 资源
  - CDN 上传文件
  - 下载企微图片 / 视频 / 文件
  - 下载个微图片 / 视频 / 文件

  ### ⚙️ 其他能力
  - 清空聊天记录、语音转文字
  - 获取微信进程 PID、当前聊天窗口信息
  - 通过 HTTP 接口启动企微（注入），支持指定端口或自动分配
  - 开关自动更新 / 自动登录 / 自动下载 / 消息免打扰

  ### 🔌 接口特性
  - **标准 RESTful HTTP API**，POST + JSON，任意语言可调用（Python / Node.js / Java / Go / PHP …）
  - 本地 Express 服务器，默认监听 `127.0.0.1:8888`
  - 按资源分模块路由：`/api/login`、`/api/message`、`/api/group`、`/api/friend`、`/api/tag`、`/api/cdn` …
  - 提供 `POST /api/call` 通用调用接口
  - 统一响应结构 `{ success, data }`，清晰错误码与 HTTP 状态码
  - 内置授权校验（按企微实例端口绑定，过期返回 403）
  - 无需登录企业微信开放平台，无需审核

  ## 🚀 快速开始

  ### 1. 下载

  前往 [Releases](https://github.com/gd122599/WeCome_work_hook/releases) 页面下载最新版客户端，解压备用。

  ### 2. 环境准备

  - 操作系统：Windows 10 / 11（64 位）
  - 企业微信版本：`v4.1.36.6012`（**请勿升级**）
  - 先登录企业微信，保持登录状态

  ### 3. 启动框架

  运行客户端，在界面中添加企微实例并完成注入。注入成功后，本地会启动 HTTP 服务，默认监听 `127.0.0.1:8888`；每个企微实例会分配一个
  **`port`**（可在界面中查看），调用 API 时需带上。

  > 也可通过 `POST /api/other/start-wecom` 以接口方式启动注入，支持指定端口或由系统自动分配。

  ### 4. 调用 API

  所有接口均为 `POST` + JSON，请求体需包含 `port`。下面以发送文本消息为例：

  ```python
  import requests

  res = requests.post("http://127.0.0.1:8888/api/message/text", json={
      "port": 6764,        # 企微实例端口（在界面中查看）
      "user_id": "788xxx", # 接收者ID 或 群ID
      "msg": "你好鸭"
  })
  print(res.json())
  # {"success": true, "data": {...}}
  ```

  ```javascript
  // Node.js
  const res = await fetch('http://127.0.0.1:8888/api/message/text', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      port: 6764,
      user_id: '788xxx',
      msg: '你好鸭'
    })
  })
  console.log(await res.json())
  ```

  ```go
  // Go
  body := `{"port":6764,"user_id":"788xxx","msg":"你好鸭"}`
  http.Post("http://127.0.0.1:8888/api/message/text", "application/json", strings.NewReader(body))
  ```

  ## ❓ 常见问题

  **Q：注入后框架没有启动或一直在登录中？**
  A：请确认企业微信版本为 `v4.1.36.6012`，其他版本暂不支持。

  **Q：调用接口报「无法连接到企微服务」或 403？**
  A：请确认企微已启动并注入成功、`port` 正确；403 通常是授权到期，请联系作者续期。

  **Q：支持哪些编程语言调用？**
  A：任何支持 HTTP 请求的语言均可，Python、Node.js、Java、Go、PHP 等都没问题。

  **Q：是否支持个人微信？**
  A：本项目专为**企业微信 PC 端**设计，不支持个人微信。
